# Documentation Standards

## README Files

All README.md files must follow these standards:

1. **Title**: Project name as H1 heading
2. **Description**: Brief overview of what the project does
3. **Installation**: Setup instructions
4. **Usage**: How to use the project
5. **License**: License information

## Markdown Format

- Use proper heading hierarchy (H1, H2, H3)
- Include code blocks with syntax highlighting
- Use lists for structured information
- Keep lines under 80 characters where possible
